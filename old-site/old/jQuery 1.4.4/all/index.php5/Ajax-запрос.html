jQuery.ajax()

Осуществляет запрос к серверу без перезагрузки страницы. Метод имеет один вариант использования:

jQuery.ajax(settings):XMLHttpRequest1.0
Это низкоуровневый метод, который используют все остальные методы, для осуществления ajax-запросов. Он имеет большое количество настроек, которые можно указать в необязательном параметре settings (он должен представлять из себя объект с настройками в формате {имя:значение, имя:значение...}). Изменить настройки по умолчанию можно с помощью метода $.ajaxSetup().

Список настроек

async

Тип:Boolean
По умолчанию:true
Описание: по умолчанию, все запросы без перезагрузки страницы происходят асинхронно (то есть после отправки запроса на сервер, страница не останавливает свою работу в ожидании ответа). Если вам понадобиться синхронное выполнение запроса, то установите параметр в false. В синхронном режиме запросы не могут быть кроссдоменными и не могут быть типа "jsonp". Имейте ввиду, что в синхронном режиме страницы могут "виснуть", пока будет выполняться запрос.

beforeSend(XMLHttpRequest)

Тип:Function
Описание: функция, которая будет вызвана сразу перед отправкой запроса. Она может быть использована для изменения объекта XMLHttpRequest непосредственно перед отправкой. Например, можно установить нужные значения header. Если функция возвратит значение false, то запрос не будет выполняться.

cache

Тип:Boolean
По умолчанию:true для всех типов данных кроме:'script' и 'jsonp'
Описание: в случае значения false браузер не будет кешировать производимый запрос.

complete(XMLHttpRequest, textStatus)

Тип:Function
Описание: функция, которая будет вызвана после завершения ajax-запроса (а так же после функций-обработчиков успешного (success) или аварийного (error) завершения запроса). В функцию передаются два параметра: объект XMLHttpRequest и значение статуса выполнения запроса (строковое значение).

contentType

Тип:String
По умолчанию:'application/x-www-form-urlencoded'
Описание: When sending data to the server, use this content-type. Default is "application/x-www-form-urlencoded", which is fine for most cases. If you explicitly pass in a content-type to $.ajax() then it'll always be sent to the server (even if no data is sent). Data will always be transmitted to the server using UTF-8 charset; you must decode this appropriately on the server side.

context

Тип:Object
Описание: позволяет указать контекст выполнения запроса. Например, можно указать в качестве контекста DOM-элемент и тогда различные обработчики ajax-запроса тоже будут выполняться в контексте данного DOM-элемента: 

$.ajax({ url: "test.html", context: document.body, success: function(){
  // в данном случае this будет равен document.body
  $(this).addClass("done");
}});
data

Тип:Object, String
Описание: данные, которые будут отправлены на сервер. Если они заданны в виде объекта, то он будет предварительно преобразован в строку, формата передачи url-параметров:fName1=value1&fName2=value2&.... В случае запроса методом GET, эта строка будет добавлена в конец url. Чтобы этого избежать, нужно изменить параметр processData (описание ниже). Заданные с помощью объекта данные, должны соответствовать формату: {fName1:value1, fName2:value2, ...}.

dataFilter(data, type)

Тип:Function
Описание: функция, с помощью которой можно обработать "сырые" данные, присланные сервером. В функцию передаются два параметра: присланные сервером, необработанные данные, а так же параметр dataType. Функция должна возвращать обработанные данные.

dataType

Тип:String
По умолчанию:выбирается автоматически, в зависимости от ситуации (xml, json, script, или html)
Описание: ожидаемый тип данных, которые должны прийти от сервера. Если он не задан, jQuery попытается определить его автоматически с помощью MIME. Опишем все доступные типы:

"xml" — будет возвращен сформированный XML-документ, который можно обрабатывать с помощью стандартных средств jQuery.
"html" — будет возвращен html, в виде обычного текста. Если он содержит скрипты в тегах <script>, то они будут исполнены, только когда html-текст будет помещен в DOM.
"script" — полученные данные будут автоматически исполнены как javascript, а потом, возвращены в виде обычного текста.
"json" — полученные данные будут автоматически исполнены как JSON, а потом, возвращены в виде объекта javascript. Начиная с jQuery 1.4, JSON проходит парсинг с более жесткими требованиями к форматированию (см. правила форматирования JSON)
"jsonp" — loads in a JSON block using JSONP. Will add an extra "?callback=?" to the end of your URL to specify the callback.
"text" — будет возвращена обычная строка текста.
error(XMLHttpRequest, textStatus, errorThrown)

Тип:Function
Описание: функция, которая будет вызвана в случае неудачного завершения запроса к серверу. Ей предоставляются три параметра: объект XMLHttpRequest, строка с описанием произошедшей ошибки, а так же объект случившегося исключения, если оно имело место. Возможные значения второго аргумента: "timeout", "error", "notmodified" и "parsererror" (в непредвиденных случаях, может быть возвращено значение null).

global

Тип:Boolean
По умолчанию:true
Описание: Отвечает за работу глобальных событий ajax-запроса (подробнее в соответствующем разделе). В случае значения false, глобальные события будут отключены для данного запроса.

ifModified

Тип:Boolean
По умолчанию:false
Описание: Allow the request to be successful only if the response has changed since the last request. This is done by checking the Last-Modified header. Default value is false, ignoring the header. In jQuery 1.4 this technique also checks the 'etag' specified by the server to catch unmodified data.

jsonp

Тип:String
Описание: Override the callback function name in a jsonp request. This value will be used instead of 'callback' in the 'callback=?' part of the query string in the url. So {jsonp:'onJsonPLoad'} would result in 'onJsonPLoad=?' passed to the server.

jsonpCallback

Тип:String
Описание: Specify the callback function name for a jsonp request. This value will be used instead of the random name automatically generated by jQuery. It is preferable to let jQuery generate a unique name as it'll make it easier to manage the requests and provide callbacks and error handling. You may want to specify the callback when you want to enable better browser caching of GET requests.

password

Тип:String
Описание: Пароль, если это требуется для выполнения запроса к серверу.

processData

Тип:Boolean
По умолчанию:true
Описание: По умолчанию, все передаваемые на сервер данные, предварительно преобразуются в строку (url-формата:fName1=value1&fName2=value2&...) и отправляются как "application/x-www-form-urlencoded". Если вам необходимо отправить данные, которые нельзя подвергать конвертированию в строку (например документ-DOM), необходимо отключить опцию processData.

scriptCharset

Тип:Strin
Описание: Only for requests with "jsonp" or "script" dataType and "GET" type. Forces the request to be interpreted as a certain charset. Only needed for charset differences between the remote and local content.

success(data, textStatus, XMLHttpRequest)

Тип:Function
Описание: Функция, которая будет вызвана после удачного завершения запроса. В функцию передаются 3 параметра: данные присланные сервером, статус выполнения запроса и объект XMLHttpRequest.

timeout(data, textStatus, XMLHttpRequest)

Тип:Число
Описание: Время ожидания ответа от сервера. Задается в в миллисекундах. Если это время будет превышено, запрос будет завершен с ошибкой и произойдет событие error (см. описание выше), которое будет иметь статус "timeout".

traditional

Тип:Boolean
По умолчанию:false
Описание: Установите значение этого параметра в true, для того, чтобы использовать традиционные параметры преобразования (сериализации).

type

Тип:String
По умолчанию:GET
Описание: Определяет тип выполняемого запроса (GET или POST). В качестве значения параметра могут использоваться PUT и DELETE, но нужно помнить, что их поддерживают не все браузеры.

url

Тип:String
По умолчанию:Адрес текущей страницы
Описание: Определяет адрес, на который будет отправлен запрос.

username

Тип:String
Описание: Имя пользователя, если это требуется для выполнения запроса к серверу.

xhr

Тип:Function
Описание: Функция, которая предоставит объект XMLHttpRequest. По умолчанию, для браузеров IE этим объектом является ActiveXObject, а в остальных случаях это XMLHttpRequest. С помощью этой функции вы можете внедрить собственную версию этого объекта.

Обработчики событий

Настройки beforeSend, error, dataFilter, success и complete (их описание есть в предыдущем разделе) позволяют установить обработчики событий, которые происходят в определенные моменты выполнения каждого ajax-запроса.

beforeSend происходит непосредственно перед отправкой запроса на сервер.
error происходит в случае неудачного выполнения запроса.
dataFilter происходит в момент прибытия данных с сервера. Позволяет обработать "сырые" данные, присланные сервером.
success происходит в случае удачного завершения запроса.
complete происходит в случае любого завершения запроса.

Пример простого использования. Выведем сообщение при удачном выполнении запроса: 

$.ajax({
  url: 'ajax/test.html',
  success: function(){
    alert('Load was performed.');
  }
});
